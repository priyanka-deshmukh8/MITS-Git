#### Introduction

# MITS Git

MITS Git is an open-source project that welcomes contributions from everyone. Whether you're a seasoned developer or new to open source, this project provides a great opportunity to get involved and see your code in action. Before you start, make sure to review our contribution guidelines. If you're unsure about how things work, don't worry! We've got you covered with answers to common questions that may arise. So buckle up and get ready for an exciting journey!

## Learn Contributing

Our comprehensive guide covers everything from creating an issue to opening a pull request. Familiarize yourself with the following key concepts and terminologies:

- **Git:** Learn what Git is and how it manages version control for your code.
- **Github:** Explore the platform where the project is hosted, enabling collaboration and version control.
- **Fork in Github:** Understand how forking allows you to create your own copy of the project.
- **Issue:** Learn what an issue is and how to create one to report problems or suggest improvements.
- **Branch:** Discover how branches help you work on different features or fixes independently.
- **Writing professional commits:** Learn best practices for writing clear and descriptive commit messages.
- **Pushing commits to your repository:** Understand how to upload your code changes to your own repository.
- **Creating a pull request to upstream:** Learn the process of submitting your changes to the main project for review.

## How it works

- MITSgit authenticates you based on your contribution file named your-github-username.mdx. To make your profile publicly visible, use the following URL format:

```
https://www.mitsgit.vercel.app/contributors/<your-github-username>
```
